git add test.txt                    #파일을 스테이징 (index에 추가)
git status --short                  #스테이징 완료 여부를 확인
git commit -m "Add test.txt"        #스테이징된 변경 커밋~오류발생: Git사용자이름/ 이메일 미설정
    ###(권장) 전역 사용자 정보 설정 — 한 번만 설정하면 이후 모든 repo에 적용
git config --global user.email "*******@gmail.com"   # GitHub 계정에 등록된 이메일
git config --global user.name "******-lee"  #실제 이름(또는 GitHub 표시명)

    ###(원하면) 이 리포지토리에서만 설정하려면 --global 없이 실행:
git config user.email "*******@gmail.com"
git config user.name "*******-lee"
    ###사용자 정보 설정 후 커밋 재시도
PS D:\2025-intel\rag-test-main> git commit -m "Add test.txt" #커밋 재시도:
    [master (root-commit) b36824e] Add test.txt
    1 file changed, 4 insertions(+)
    create mode 100644 test.txt
git log -1 --oneline    #커밋 확인(선택):
    b36824e (HEAD -> master) Add test.txt
  — 커밋이 성공적으로 만들어진 것을 확인 
        ==>다음은 원격(remote) 확인 및 푸시 단계
git remote -v   #원격이 있는지 먼저 확인  ~ 없음 확인 완료
    ###로컬에 원격 추가:
git remote add origin https://github.com/upendi-lee/rag-test.git
    ###원격이 잘 등록됐는지 확인:
git remote -v
    ###원격에 푸시 (현재 브랜치는 master이므로 master에 푸시):
git push -u origin master

원격으로 푸시가 성공했고 로컬 master 브랜치가 origin/master와 연결된 상태입니다.
이제 원격에 제대로 올라갔는지 검증

    ✅# GitHub에 아직 원격(빈 리포)이 없는 경우

GitHub 웹에서 새 리포지토리 만들기

https://github.com/new 로 가서 리포지토리 이름(예: rag-test-main) 입력 → 생성
(웹에서 표시되는) 리모트 URL 복사 후 A) 단계의 명령으로 추가하고 푸시:
git remote add origin https://github.com/YOUR_ACCOUNT/YOUR_REPO.git
git push -u origin master

    ✅### 검증 방법(원하시는 방법으로 하나씩 실행
git status           # 워킹 트리 상태
git branch -vv       # 브랜치가 origin/master를 추적하는지 확인
git log --oneline -5 # 최근 커밋 확인

-----------------------------------------
✅✅✅[신규 추가 및 업데이트]✅✅✅

✅1.로컬 상태 확인 — 어떤 파일이 변경/추가/삭제됐는지 확인합니다:
git status --short

✅2.변경 전체를 스테이징 (모든 변경 포함) — 특정 파일만 올리고 싶으면 그 파일들만 대신 지정하세요:
git add -A
(특정 파일만: git add README.md package/ 파일경로 등)

✅3.스테이징된 내용을 확인:
git status --short

✅4.커밋 (작업 내용을 설명하는 메시지로 바꿔주세요):
git commit -m "Update docs and package files"
(이미 같은 변경을 커밋한 경우 메시지를 바꾸거나 --amend 등 옵션 사용할 수 있음 — 원하면 안내해 드립니다.)

✅5.원격으로 푸시 (현재 master가 origin/master를 추적 중이므로 일반적으로 바로 푸시됨):
git push

✅6.푸시 결과 확인(선택):
git log --oneline -5
git branch -vv

✅추가 안내(원하면):

별도 브랜치로 작업해서 PR로 올리려면:
git checkout -b feature/your-topic
git push -u origin feature/your-topic

실행하신 후 출력 결과확인. ✅
